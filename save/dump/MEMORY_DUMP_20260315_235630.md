## Key Architecture Notes

### WebSocket ConnectionManager (websocket.py)
- `_stream_owners: dict[str, WebSocket]` - maps stream_id to initiating client
- `_batch_buffers: dict[str, list[str]]` - pending chunks per stream
- `_batch_tasks: dict[str, asyncio.Task]` - flush timers per stream
- Stream events filtered by type: `stream_start`, `stream_chunk`, `stream_end`, `stream_cancelled`, `filler_trigger`
- Non-stream events (thinking, backend_log) broadcast to all

### Incremental Indexing (rag_service.py + knowledge_graph_service.py)
- `kg_index_state` table with `content_type` (unique) + `last_indexed_at`
- Content types: `fts5_note`, `fts5_transcript`, `fts5_file_context`, `fts5_project_file`, `fts5_document`, `kg_document`, `kg_transcript`, `kg_note`
- First run: watermark defaults to year 2000 (indexes everything)
- Subsequent runs: only items with created_at/updated_at > watermark

### LLM Client Cache (llm_service.py)
- Cache key format: `{provider}:{base_url}:{api_key[:8]}`
- Singletons per unique key combination
- Covers OpenAI, AsyncOpenAI, Anthropic clients
